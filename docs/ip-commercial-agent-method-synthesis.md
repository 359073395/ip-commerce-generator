# IP商业内容 Agent 方法总纲

生成时间：2026-07-10

## 一句话结论

这套系统的核心不是“帮用户写文案”，而是把用户的行业、受众、证据、产品、转化入口，套进一套经过成功账号和对标账号验证的个人 IP 商业内容方法库里，自动生成定位、选题、脚本、画面和成交承接。

## 母方法

吾天律师账号提供的是个人 IP 的母方法，不只适用于工程律师：

1. 先找最痛的受众，而不是先找内容形式。
   工程老板不是泛泛的“工程人”，而是被工程款卡住、上面不结账、下面还要付采购和工资的人。

2. 先放大信任资产，而不是先讲专业知识。
   金额、胜诉、强对手、案件数量、客户身份、复杂程度、本地关系感，都是“非金钱炫富”。

3. 先替用户说出心里话，再给解决路径。
   工程老板怕失去关系、怕得罪国央企、怕官司赢了也拿不到钱，所以内容必须讲利益、讲现实、讲关系失效、讲行业悲观。

4. 先用真实人设过滤假流量。
   方言、江湖气、情绪、酒后状态、脏话感、对话、参观、团建，都不是随意表演，而是在证明这个人真实、有情绪、站在客户一边。

5. 先保留爆款结构，再替换行业事实。
   可借鉴同行爆款、低粉爆款、商业认知和人性内容。保留开头、句式、注意力顺序和情绪结构，替换为自己的案例、行业词、口吻和承接。

6. 先给低阻力转化动作。
   对高客单专业服务，评论少不等于差；真正高意向信号是私信、电话、要联系方式、问“是不是吾天律师”、感同身受。

## 对标账号验证出的通用结构

所有行业可以归成同一条链路：

注意力钩子 -> 身份/证据 -> 用户痛点 -> 结果证明 -> 异议化解 -> 转化入口

不同账号只是把“证据”换成行业可见资产：

| 行业类型 | 信任资产 | 常用钩子 | 转化入口 |
| --- | --- | --- | --- |
| 工程律师/专业服务 | 金额、胜诉、团队、执行回款、强对手 | 行业现实、客户困境、风险共担 | 电话、私信、线下介绍 |
| 美业/变美 | 前后对比、忙碌过程、千元客单、顾客反应 | 改造结果、第一视角、街头挑战 | 到店、预约、课程 |
| 知识付费/咨询 | 强观点、学员结果、老板焦虑、人性洞察 | 反常识、认知差、低粉爆款结构 | 私信关键词、资料、课程 |
| 装修/家居 | 现场、流程、材料、验收标准、避坑清单 | 数字清单、你就这么问、内幕 | 咨询、到店、报价 |
| 宠物/测评 | 风险后果、测评标准、预算建议、产品对比 | 误区纠正、风险提醒、一句话总结 | 商品、直播、到店 |
| 工厂老板/B端 | 工厂、展厅、设备、库存、投入金额 | 规模前置、老板故事、供应链证明 | 询盘、直播、合作 |

## Agent 调用规则

当用户在前端输入项目资料时，Agent 应按这个顺序工作：

1. 判断账号类型。
   个人 IP、IP 带服务、IP 带产品、带货、同城门店、B 端工厂，先分流。

2. 判断商业目标。
   涨粉、咨询、私信、电话、到店、课程成交、产品成交、招商询盘，不能混在一起写。

3. 检索方法卡。
   优先命中母方法卡，再命中行业对标卡，再命中视频级方法卡。

4. 生成完整骨架。
   前端可以简单，但后端输出必须完整：定位、受众、痛点、信任资产、选题矩阵、脚本骨架、画面建议、CTA、复盘指标。

5. 做质量自检。
   如果结果没有体现前端选择、没有结合知识库方法、没有给行业证据、没有转化入口，就必须降分并触发修复。

## 证据边界

当前视频级拆解主要来自抖音页面标题、简介、可见文本、封面、评论片段和平台章节摘要。没有逐字稿的视频，不编造原话。后续如果继续补强，最有价值的是补充高咨询视频的完整口播稿、私信第一句话、电话咨询反馈和真实成交数据。
