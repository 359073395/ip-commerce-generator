## 一句话结论

教知识不是把专业内容原样搬到视频里，而是把用户当下有需求、能听懂、会保存、愿意继续关注的知识讲出来。它的商业价值是：用知识证明专业，用专业建立信任，用信任承接成交。

## 教知识在个人IP带货里的作用

课程把教知识放在脚本体系里，是因为它能解决三个问题：

- 给用户一个关注你的理由：你持续帮他解决问题。
- 给用户一个信任你的理由：你能把复杂问题讲明白。
- 给用户一个成交理由：你不仅懂，还能帮他做选择、避坑、降低成本、变得更好。

如果一个账号只会展示产品或讲故事，但没有知识输出，用户很难判断你到底专业在哪里。教知识就是把专业能力显性化。

## 知识脚本的误区

错误理解：

- 把课堂、培训、咨询里的内容原封不动搬到短视频。
- 只讲自己想讲的专业术语，不考虑用户是否有需求。
- 觉得“干货越多越好”，最后信息太满，用户反而记不住。
- 只追求收藏，但不引导关注和后续转化。

正确理解：

- 知识要从目标人群的需求出发。
- 信息差不等于高深知识，用户没听过、当下用得上，就有价值。
- 知识内容要能让用户产生“这个人懂我、也懂这个行业”的感觉。
- 知识脚本也要有钩子、结构和结果，不是随口讲课。

## 先确定目标人群

课程里反复强调：需求是由人群决定的。

同一个知识，对不同人群价值完全不同：

- “去医院看病流程”太泛。
- “第一次去上海第九人民医院看某类病怎么少走弯路”就更精准。
- “坐飞机遇到突发下降怎么办”对经常飞的人可能普通，对第一次坐飞机的人就很有用。

做知识脚本时，先问：

- 这条视频具体讲给谁听？
- 他现在有什么困惑、焦虑、目标或即将面对的场景？
- 我讲完以后，他能立刻做什么？

## 知识内容的三个优势

### 1. 粉丝更精准

知识脚本吸引来的不是泛流量，而是对某个问题、行业、产品或结果感兴趣的人。这样的粉丝虽然可能涨得慢，但更容易成为咨询、成交或复购对象。

### 2. 创作更简单

知识内容不一定需要复杂场景。很多时候，一段清楚的表达、一个清单、一个表格、一个案例拆解，就能完成一条视频。

### 3. 更容易建立专业身份

当用户反复看到你能解释问题、拆解选择、指出风险，他会把你和这个领域绑定起来。个人IP真正的符号不是一句口号，而是用户记住“你这个人”。

## 五类容易成立的知识选题

### 1. 追求更好

帮助用户变得更好、更美、更会赚钱、更会表达、更有效率、更有面子。

例子：

- 怎么让账号更容易接到咨询。
- 怎么让穿搭更显瘦。
- 怎么让装修预算更值。
- 怎么从普通文案改成更容易成交的文案。

这类内容的吸引力来自向往感：用户想变好，所以愿意听。

### 2. 解决难题

用户现在遇到问题，或者很快就会遇到问题，你给出可执行解法。

例子：

- 客户来了但不成交怎么办。
- 第一次装修不知道怎么选材料怎么办。
- 视频有播放但没有咨询怎么办。
- 去某个机构办事如何少排队、少踩坑。

这类内容最容易引发点赞和关注，因为用户希望你持续帮他解决问题。

### 3. 避坑防风险

提醒用户哪些选择会造成损失、麻烦、低效或后悔。

例子：

- 新手做账号最容易踩的3个坑。
- 买某类产品前必须确认的5件事。
- 装修合同里最容易被忽略的条款。
- 投流前不要先犯这几个错误。

这类内容适合和真实案例结合，可信度会更强。

### 4. 降低成本

教用户在不降低核心体验和关键质量的前提下，少花冤枉钱。

课程里的重点是：降低成本不能等于降低用户体验，也不能把产品的关键要素砍掉。

例子：

- 大牌同类成分怎么判断是否值得买。
- 装修预算有限，哪些地方能省，哪些地方不能省。
- 投放预算有限，先测什么内容最划算。
- 选品时如何避免花钱买到无效库存。

### 5. 丰富谈资

给用户一些能在社交、工作、消费决策里用得上的知识，让他觉得“我多知道了一点”。

例子：

- 一个行业里的冷知识。
- 一类产品背后的判断标准。
- 不同品牌/地区/价位的差异。
- 让用户显得更懂行的入门常识。

这种内容不一定立刻成交，但能扩大账号的认知面。

## 推荐/清单型知识

本节后半段重点讲到“推荐型”内容：给用户列出可选择对象，并说明为什么。

推荐型不是简单列清单，而是要有判断标准：

- 为什么选这个？
- 它适合谁？
- 它的优势是什么？
- 使用时有什么注意点？
- 和其他选择相比差在哪里？

推荐型内容适合做成表格、清单、分级、排序。它很容易触发收藏，因为用户会觉得“以后可能用得上”。

## 知识脚本的排序原则

不要把最熟悉、最基础、最像说明书的东西都堆在前面。课程里强调，顺序会影响用户是否继续看。

可用顺序：

1. 先抛出用户最关心的结果、风险或场景。
2. 再给出关键判断标准。
3. 中间放具体步骤、清单或表格。
4. 最后补充产品、服务、关注理由或成交入口。

如果直接先讲产品，用户容易觉得你在卖；如果先讲用户的困惑和判断方法，用户更容易接受你的专业。

## 知识脚本和成交的关系

一条知识视频的理想路径：

```text
用户觉得有用
-> 点赞/收藏/评论/转发
-> 继续看到你
-> 关注你
-> 认可你的专业
-> 在需要时买你的产品或服务
```

所以知识脚本不能只追求“这条讲得对”，还要让用户愿意留下关系。

## 可复用脚本模板

```text
开头：
如果你是[目标人群]，最近正在[具体场景/问题]，
先别急着[错误动作]，先看这[数量]个判断标准。

主体：
第一点：[标准/误区/步骤]
为什么重要：[原因]
怎么做：[动作]

第二点：[标准/误区/步骤]
为什么重要：[原因]
怎么做：[动作]

第三点：[标准/误区/步骤]
为什么重要：[原因]
怎么做：[动作]

结尾：
我把这套[清单/表格/流程]整理好了，
后面继续讲[下一步问题/进阶选题]。
```

## 行动清单

- 每个行业先写出20个目标人群的真实问题。
- 每个问题改写成“追求更好、解决难题、避坑、降低成本、丰富谈资”中的一种。
- 每条知识视频只讲一个明确结论，不贪多。
- 尽量给清单、表格、步骤，让用户有收藏理由。
- 把产品或服务放在知识之后，让用户先认可你的判断。
