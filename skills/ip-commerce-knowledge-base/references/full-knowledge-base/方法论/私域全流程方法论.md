# 私域全流程方法论

## 核心逻辑

私域的价值不在于“把人加到微信”，而在于用持续触达、信任铺垫、销售路径、服务记录和复购机制，把一次流量变成长期客户资产。

## 标准流程

```text
调研对标 -> 养号准备 -> 朋友圈布局 -> 公域引流 -> 私域承接 -> 销售SOP -> 成交 -> 二转服务 -> 复购 -> 售后 -> 里程碑沉淀
```

## 关键动作

### 1. 调研对标

至少拆 10 个同行，记录账号、产品、价格、核心观点、私域入口、朋友圈和社群承接方式。

### 2. 朋友圈前置

客户进来前，朋友圈要先完成信任建设：

- 我是谁。
- 我能解决什么问题。
- 为什么你应该找我。
- 我的客户反馈和过程证据。

### 3. 引流一致性

公域内容讲什么，私域第一句就接什么。不要让客户从直播/短视频进入私域后立刻感到主题断裂。

### 4. 销售 SOP

先用 SOP 降低错误率，再通过每日复盘补细节：

- 客户常见问题。
- 未成交原因。
- 成功聊天记录。
- 产品体验表达。
- 客户异议处理。

### 5. 二转服务

成交后要有平滑过渡，不能让客户觉得“卖完就被转手”。二转人员必须看到客户档案，知道客户来源、购买记录、核心问题和下一步需求。

### 6. 复购设计

复购来自持续服务和需求挖掘：

- 产品线更新。
- 客户问题分类。
- 精准社群运营。
- 积分/权益机制。
- 生日、节日、关键节点维护。

### 7. 售后管理

售后前置说明规则，售后发生时先安抚情绪，再处理问题。售后处理不好，会伤害后续复购和口碑。

### 8. 项目里程碑

每个项目保留一份沉淀文档，记录有效动作、踩坑、活动复盘、表格、话术和对接流程，方便下一个项目复制。

## 一句话复盘

私域全流程的本质，是把“流量进入”变成“关系推进”，再把“第一次成交”变成“长期服务和复购”。
