# Continuing Learning Workflow

Use this when the user gives new folders, videos, documents, tables, PDFs, screenshots, or course materials and says to 学习, 继续学习, 入库, 建知识库, 更新知识库, or summarize learned materials.

## Learning Principles

1. Learn into two categories only:
   - `个人IP`
   - `带货视频`
2. Do not make course names the final classification.
3. Prefer full learning over sampling. Sampling is only for quick preview, not the final knowledge base, unless the user explicitly accepts sampling.
4. Preserve source traceability, but synthesize knowledge into reusable workflows, templates, scripts, and review logic.
5. After learning, update master handbooks or supporting method libraries, not just raw notes.

## File Inventory

Before processing, list file types and counts:

- video/audio
- docx/doc/pdf/ppt/pptx
- xls/xlsx/csv
- images/screenshots
- subtitles/transcripts
- unreadable or damaged files

Use fast local search/listing tools first. Record unreadable files instead of stopping the whole run.

## Video and Audio

For video/audio material:

1. Build a manifest with path, filename, size, duration if discoverable.
2. Transcribe fully when possible.
3. If full transcription is expensive or blocked, preview a sample only to understand scope, then explain the limitation and continue with available text.
4. Convert transcripts into reusable notes:
   - concept
   - formula
   - workflow
   - script card
   - example
   - execution checklist
   - review metric

Store or update transcript/notes in the knowledge base if the existing folder structure supports it.

## Documents, Spreadsheets, PDFs, Images

For documents:

- Convert DOCX/PPTX/PDF/XLSX to readable Markdown or tables using appropriate tooling.
- Preserve tables when they are templates, prompts, scripts, or checklists.
- OCR image-heavy materials when needed.

Extract:

- positioning frameworks
- scripts
- prompt templates
- content calendars
- account setup rules
- live room/small-cart techniques
- product selection logic
- data review tables
- examples that can be adapted

## Synthesis Format

For every learned source batch, synthesize into:

```text
1. What new knowledge was learned
2. Which category it belongs to: 个人IP / 带货视频 / both
3. What workflow or formula should be added
4. What templates or examples should be reused
5. What files were updated
6. What could not be read
```
